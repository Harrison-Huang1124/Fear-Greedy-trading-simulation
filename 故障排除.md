# 无法打开 http://localhost:3000 的故障排除指南

1. ✅ Node.js 已安装 (v22.17.1)
2. ✅ npm 已安装 (10.9.2)
3. ✅ 依赖包已安装 (npm install 已完成)

## 🚀 现在启动应用

### 方法1：使用启动脚本（推荐）

双击运行 `start_react.bat` 文件

### 方法2：手动启动

1. 打开命令提示符（CMD）或 PowerShell
2. 进入项目目录：
   ```bash
   cd D:\ISEF
   ```
3. 启动React开发服务器：
   ```bash
   npm start
   ```

## ⚠️ 常见问题

### 问题1：端口3000已被占用

**错误信息**：
```
Something is already running on port 3000
```

**解决方法**：
1. 关闭占用3000端口的程序
2. 或使用其他端口：
   ```bash
   set PORT=3001
   npm start
   ```
   然后访问 `http://localhost:3001`

### 问题2：找不到模块错误

**错误信息**：
```
Cannot find module 'react'
```

**解决方法**：
```bash
npm install
```

### 问题3：React脚本启动失败

**错误信息**：
```
'react-scripts' is not recognized
```

**解决方法**：
```bash
npm install react-scripts --save
```

### 问题4：浏览器显示空白页面

**可能原因**：
- 后端Flask服务器未启动
- API连接失败

**解决方法**：
1. 确保Flask后端正在运行：
   ```bash
   python app.py
   ```
2. 检查 `http://localhost:5000/api/stocks` 是否可以访问
3. 查看浏览器控制台（F12）的错误信息

### 问题5：编译错误

**检查**：
1. 打开浏览器控制台（F12）
2. 查看错误信息
3. 检查终端中的错误输出

## 📋 启动检查清单

在启动前，请确认：

- [ ] Node.js 已安装（运行 `node --version`）
- [ ] npm 已安装（运行 `npm --version`）
- [ ] 依赖已安装（`node_modules` 文件夹存在）
- [ ] Flask后端正在运行（`python app.py`）
- [ ] 端口3000未被占用

## 🔍 验证步骤

1. **检查依赖安装**：
   ```bash
   dir node_modules
   ```
   应该看到很多文件夹

2. **检查Flask后端**：
   在浏览器访问：`http://localhost:5000/api/stocks`
   应该返回股票列表

3. **启动React**：
   ```bash
   npm start
   ```
   应该看到：
   ```
   Compiled successfully!
   You can now view neurotrade-fgi-dashboard in the browser.
   Local:            http://localhost:3000
   ```

4. **自动打开浏览器**：
   React会自动打开浏览器，如果没有，手动访问 `http://localhost:3000`

## 🆘 如果仍然无法打开

1. **检查防火墙**：确保防火墙没有阻止Node.js

2. **检查杀毒软件**：某些杀毒软件可能阻止Node.js

3. **尝试其他端口**：
   ```bash
   set PORT=3001 && npm start
   ```

4. **清除缓存并重新安装**：
   ```bash
   rmdir /s /q node_modules
   del package-lock.json
   npm install
   npm start
   ```

5. **查看详细错误**：
   - 打开浏览器控制台（F12）
   - 查看Network标签页
   - 查看终端中的完整错误信息
